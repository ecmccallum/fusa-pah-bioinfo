Objective

Assemble cleaned and decontaminated Illumina paired-end reads into a de novo draft genome for Fusarium solani using SPAdes in isolate mode. The resulting scaffolds will be used as input for gene annotation in Stage 5.

Troubleshooting log

Attempts 1–4 — OOM kills on standard partition

Four successive SLURM jobs (IDs 1467968, 1471686, 1471906, 1476604) failed with OS return value -9 (SIGKILL). SLURM logs confirmed out-of-memory kills via cgroup handler. Jobs were submitted to the standard partition with 64G reserved and SPAdes told to use 48G — insufficient for the scaffolding step.

slurmstepd: error: Detected 1 oom-kill event(s) in step XXXXXX.batch cgroup. Some processes may have been killed by the cgroup out-of-memory handler.

Crucially, k-mer graph construction at K21, K33, K55, and K77 completed successfully across these runs and was preserved on disk.

Resolution — midmem partition with restart

Identified the midmem partition (244G nodes) as the appropriate resource. Required specifying both --account=midmem and --qos=lessmidmem_qos in the SLURM header — partition alone was insufficient. Used --restart-from last to resume from the completed K77 checkpoint, skipping all graph construction.

Submitted batch job 1563985 — partition: midmem, status: completed successfully.

Final SLURM script

#!/bin/bash
#SBATCH --job-name=spades_fusarium
#SBATCH --output=/zfs/home/22/emccallum/fusarium_pah/logs/spades_%j.log
#SBATCH --error=/zfs/home/22/emccallum/fusarium_pah/logs/spades_%j.err
#SBATCH --time=5-00:00:00
#SBATCH --mem=200G
#SBATCH --cpus-per-task=8
#SBATCH --partition=midmem
#SBATCH --account=midmem
#SBATCH --qos=lessmidmem_qos

source /softs/contrib/apps/anaconda/3/etc/profile.d/conda.sh
conda activate fusarium_env

spades.py --restart-from last \
  -o ~/fusarium_pah/results/assembly \
  --threads 8 \
  --memory 180

echo "SPAdes finished at $(date)"

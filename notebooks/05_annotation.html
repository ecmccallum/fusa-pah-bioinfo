5.1 Environment Setup

Funannotate was not available as a Pyrene module or within the existing fusarium_env environment. A dedicated environment was created using mamba rather than conda. Mamba uses a C++ dependency solver and is substantially faster for large dependency trees — conda's solver is prone to hanging on problems of this size (~100 packages). 

mamba create -n funannotate_env -c conda-forge -c bioconda funannotate
conda activate funannotate_env
export FUNANNOTATE_DB=/home/22/emccallum/funannotate_db

The FUNANNOTATE_DB variable must be set in every session and is included in all SLURM scripts for this stage.

5.4 funannotate clean — SLURM job 1568332

The clean step removes degenerate contigs containing only 1–3 nucleotides (SPAdes assembly artefacts), renames contig headers to short standard identifiers, and uses minimap2 to identify duplicate contigs among scaffolds shorter than the N50. 

funannotate clean \
  -i data/assembly/fusarium_final_assembly.fasta \
  -o results/annotation/funannotate/fusarium.clean.fasta

Run as SLURM job 1568332, standard partition, node n005, 8 CPUs, 24G RAM. The minimap2 duplication check across 4,282 sub-N50 contigs took approximately 1.5 hours.

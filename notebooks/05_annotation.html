Environment Setup

Funannotate was not available in the existing fusarium_env and was not found on the active PATH, so a dedicated environment was created using mamba rather than conda. Mamba uses a C++ dependency solver and is substantially faster for large dependency trees — funannotate has approximately 100 dependencies and conda's solver is prone to hanging on problems of this size. 

mamba create -n funannotate_env -c conda-forge -c bioconda funannotate
conda activate funannotate_env

The following environment variables are required and set permanently in ~/.bashrc:

export FUNANNOTATE_DB=/home/22/emccallum/funannotate_db
export GENEMARK_PATH=/home/22/emccallum/gmes_linux_64_4
export PATH=/home/22/emccallum/gmes_linux_64_4:$PATH

5.1 funannotate clean — SLURM job 1568332

The clean step removes degenerate contigs containing only 1–3 nucleotides (SPAdes assembly artefacts) and uses minimap2 to identify and remove duplicate contigs among scaffolds shorter than the N50. Contig header renaming is handled by the subsequent sort step. 

funannotate clean \
  -i data/assembly/fusarium_final_assembly.fasta \
  -o results/annotation/funannotate/fusarium.clean.fasta

5.2 funannotate sort

The sort step renames contig headers from verbose SPAdes identifiers (e.g. NODE_1_length_5050286_cov_47.3) to short standardised identifiers (scaffold_1, scaffold_2, etc.) and sorts contigs by length. Many downstream tools cannot handle long or special-character contig names. A known bug in funannotate v1.8.17 causes a TypeError if --minlen is not explicitly passed; fixed with --minlen 0. 

funannotate sort \
  -i results/annotation/funannotate/fusarium.clean.fasta \
  -o results/annotation/funannotate/fusarium.sorted.fasta \
  -b scaffold \
  --minlen 0

Run interactively on login node. Completed in under one minute.

Sort verdict. 4,308 contigs sorted by length and renamed scaffold_1 … scaffold_4308.

5.3 funannotate mask — SLURM job 1572174

The mask step identifies repetitive regions (transposable elements, simple repeats) and soft-masks them with lowercase letters. Gene predictors ignore lowercase-masked regions, preventing false gene calls within repeat elements. Funannotate uses tantan for soft-masking against the installed repeat database. 

funannotate mask \
  -i results/annotation/funannotate/fusarium.sorted.fasta \
  -o results/annotation/funannotate/fusarium.masked.fasta \
  --cpus 8

5.4 funannotate predict — SLURM job 1572453

The predict step is the core gene-finding stage. Funannotate runs multiple ab initio predictors — Augustus, GeneMark-ES, SNAP, GlimmerHMM, and CodingQuarry — and uses EvidenceModeler to produce a consensus gene set. Gene models are assigned the prefix FUS.

Note on flag usage: -s sets the organism label written into output files. --augustus_species and --busco_seed_species independently specify which pre-trained models to use for prediction — F. graminearum is the closest available trained model to F. solani. 

funannotate predict \
  -i results/annotation/funannotate/fusarium.masked.fasta \
  -o results/annotation/funannotate/predict \
  -s "Fusarium solani" \
  --augustus_species fusarium_graminearum \
  --busco_seed_species fusarium_graminearum \
  --cpus 16 \
  --name FUS

SLURM job 1572453 — standard partition, 16 CPUs, 32G RAM, 48 hour time limit. Status: pending → running.

Results pending. Job 1572453 submitted and running. Output will include a GFF3 gene annotation file and a predicted protein FASTA in results/annotation/funannotate/predict/. Expected gene count for a Fusarium solani genome: approximately 12,000–15,000 gene models.

5.5 funannotate annotate — Pending

Once predict is complete, the annotate step will assign functional descriptions to each predicted protein by searching against all installed Funannotate databases: Pfam, InterPro, UniProt/Swiss-Prot, dbCAN, MEROPS, GO, MiBIG, and gene2product. eggNOG mapper will not be included due to network restrictions on Pyrene preventing database download. The annotate step produces the final GFF3 and protein FASTA files used in Stage 6.

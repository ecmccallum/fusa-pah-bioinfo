Environment Setup

Funannotate was not available in the existing fusarium_env and was not found on the active PATH, so a dedicated environment was created using mamba rather than conda. Mamba uses a C++ dependency solver and is substantially faster for large dependency trees — funannotate has approximately 100 dependencies and conda's solver is prone to hanging on problems of this size. 

mamba create -n funannotate_env -c conda-forge -c bioconda funannotate
conda activate funannotate_env

The following environment variables are required and set permanently in ~/.bashrc:

export FUNANNOTATE_DB=/home/22/emccallum/funannotate_db
export GENEMARK_PATH=/home/22/emccallum/gmes_linux_64_4

5.2 funannotate sort

The sort step renames contig headers from verbose SPAdes identifiers (e.g. NODE_1_length_5050286_cov_47.3) to short standardised identifiers (scaffold_1, scaffold_2, etc.) and sorts contigs by descending length. Many downstream tools cannot handle long or special-character contig names. A known bug in funannotate v1.8.17 causes a TypeError if --minlen is not explicitly passed; fixed with --minlen 0. 

funannotate sort \
  -i results/annotation/funannotate/fusarium.clean.fasta \
  -o results/annotation/funannotate/fusarium.sorted.fasta \
  -b scaffold \
  --minlen 0

Run interactively on login node. Completed in under one minute.

Sort verdict. 4,308 contigs sorted by length and renamed scaffold_1 … scaffold_4308.

5.3 funannotate mask — SLURM job 1572174

The mask step identifies repetitive regions (transposable elements, simple repeats) using tantan and soft-masks them with lowercase letters. Soft-masking marks repetitive regions in lowercase so downstream gene predictors can treat these regions differently and reduce false gene calls in repetitive sequence. 

funannotate mask \
  -i results/annotation/funannotate/fusarium.sorted.fasta \
  -o results/annotation/funannotate/fusarium.masked.fasta \
  --cpus 8

5.4 funannotate predict

The predict step is the core gene-finding stage. Funannotate runs multiple ab initio predictors — Augustus, GeneMark-ES, SNAP, GlimmerHMM, and CodingQuarry — trains gene models using BUSCO outgroups, and uses EvidenceModeler (EVM) to produce a consensus gene set weighted by predictor reliability. Gene models are assigned the locus tag prefix FUS.

Note on flags: -s sets the organism label written into output files. --augustus_species and --busco_seed_species independently specify which pre-trained models to use — F. graminearum is the closest available trained species in the Augustus database. --protein_evidence passes external protein sequences to EVM to provide additional support for evaluating gene models.

5.5 funannotate annotate — Running

Following successful completion and quality assessment of predict v2, funannotate annotate was submitted to assign functional descriptions to the predicted gene models. This step searches the predicted proteins against the installed funannotate databases to assign Pfam domains, GO terms, MEROPS protease families, dbCAN carbohydrate-active enzyme families, MiBIG biosynthetic gene cluster matches, UniProt/Swiss-Prot descriptions, and gene2product names. 

funannotate annotate \
  -i results/annotation/funannotate/predict_v2 \
  --cpus 16 \
  --busco_db dikarya

Standard partition, 16 CPUs, 32G RAM, 24 hour time limit. --busco_db dikarya matches the dikarya_odb9 lineage used internally during predict.

InterPro XML vs InterProScan — important distinction. The funannotate database includes an InterPro XML file (v108.0) which provides cross-reference data for domain naming. This is not equivalent to running InterProScan. InterProScan performs active domain searches against Pfam, PANTHER, TIGRFAM, and other member databases using the query protein sequences. InterProScan is not installed on Pyrene and represents a significant gap in this annotation. Resolution with the postdoc is required before Stage 6 analysis is considered complete.
eggNOG excluded. eggNOG-based orthology annotation is not included in this annotate run due to the confirmed permanent database access block at cluster level. This means GO term and KEGG pathway assignments from orthology mapping will be absent from the annotation output.

Output will be written to results/annotation/funannotate/predict_v2/annotate_results/ upon completion. Expected output files include annotated GFF3, GenBank, protein FASTA, transcript FASTA, and a functional annotation summary. These will form the primary input for candidate PAH-relevant enzyme screening in Stage 6.

Conclusions

Gene prediction is complete. The v2 predict run produced 22,338 predicted protein-coding gene models and 341 tRNAs from a 57.4 Mb assembly. BUSCO scores of 98.3% complete at genome level and 97.0% complete at protein level, with low duplication, indicate that the assembly and conserved gene space are highly complete. The 22.7% exon-protein evidence overlap represents a 7.5-fold improvement over v1 and indicates substantially improved protein support for the v2 gene models. The total gene count remains higher than expected for many published Fusarium genomes and should be discussed with the postdoc, but the BUSCO results argue against major assembly duplication or a catastrophic annotation failure. Functional annotation via funannotate annotate is currently running, and its outputs will be used for candidate PAH-relevant enzyme screening in Stage 6.
